🗄️ The SQL Database

A relational database modeling the business, with 21 analytical queries answering real questions.

The same data also lives in a SQLite database. This is the part that shows real data skills: a designed schema, two related tables, and a library of queries that turn rows into insight. Everything is plain text you can read in the sql/ folder.

The design (two related tables)

TableHolds
vehiclesOne row per bike/car: make, model, year, buy/sell price, profit, dates, status, buyer.
partnersOne row per partner per vehicle: who, how much cash, what labor — linked back to a vehicle by id.

This one-to-many link (a vehicle can have several partners) is what lets the queries split contributions by person using a JOIN.

What the 21 queries answer

ThemeExamples
Headline KPIsTotal profit, ROI on cost, average days to sell
TrendsProfit by month, by make, by model
OperationsInventory aging, speed-vs-profit, top & bottom flips
PartnersCash contributed per partner (via JOIN)

A sample query

SELECT make || ' ' || model AS model_name,
       COUNT(*)            AS times_sold,
       ROUND(AVG(days_to_sell),1) AS avg_days,
       ROUND(AVG(profit))        AS avg_profit
FROM vehicles
WHERE status = 'Sold'
GROUP BY model_name
ORDER BY times_sold DESC, avg_days ASC;

→ Result: the Kawasaki Ninja 400 is the best mover — 6 sold, ~11 days each.
